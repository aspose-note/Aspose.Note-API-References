## Page.IsConflictPage property

Bu sayfanın bir çakışma sayfası olup olmadığını belirten bir değer alır veya ayarlar.

```csharp
public bool IsConflictPage { get; set; }
```

### Notlar

İki kullanıcı aynı içeriği güncellemeye çalıştığında çakışma sayfası ortaya çıkar. Bu durumda ilk kullanıcının değişiklikleri her zamanki gibi yazılır. Ancak başka bir kullanıcının değişiklikleri birleştirilemez. Yani sadece sayfanın bir kopyası oluşturulur. ve çakışma olarak işaretlendi.

Bu sürümde, çakışmalar ilk kullanıcının değişiklikleri lehinde çözülür. Dolayısıyla, belgede çakışma sayfaları varsa, bunlar geçmişte gösterilir ancak kaydetme sırasında atlanırlar. Bu sayfaları kaydetmek için bu bayrağı sıfırlamak mümkündür. her zamanki gibi tarihte.

Çakışmaya göre işleme sayfasının ayrıntılı örneği çevrimiçi belgelerde bulunabilir.

### Örnekler

Bir sayfanın çakışma sayfası olup olmadığının nasıl kontrol edileceğini gösterir (yani, OneNote'un otomatik olarak birleştiremeyeceği değişikliklere sahiptir).

```csharp
string dataDir = RunExamples.GetDataDir_Pages();

// OneNote belgesini yükle
Document doc = new Document(dataDir + "Aspose.one", new LoadOptions { LoadHistory = true });

var history = doc.GetPageHistory(doc.FirstChild);
for (int i = 0; i < history.Count; i++)
{
    var historyPage = history[i];
    Console.Write("    {0}. Author: {1}, {2:dd.MM.yyyy hh.mm.ss}",
                    i,
                    historyPage.PageContentRevisionSummary.AuthorMostRecent,
                    historyPage.PageContentRevisionSummary.LastModifiedTime);
    Console.WriteLine(historyPage.IsConflictPage ? ", IsConflict: true" : string.Empty);

    // Varsayılan olarak çakışan sayfalar kaydedilirken atlanır.
    // Çakışmaz olarak işaretlerseniz, tarihte her zamanki gibi kaydedilecektir.
    if (historyPage.IsConflictPage)
        historyPage.IsConflictPage = false;
}

doc.Save(dataDir + "ConflictPageManipulation_out.one", SaveFormat.One);
```
